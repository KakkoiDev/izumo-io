# Using an AI Coding Assistant

- [A01: Safety, Privacy & Mindset](lessons/a01.md) - Before you install anything: what AI is, how it fails, and the three rules that keep you safe. AI is a power tool, not a friend.
- [A02: Terminal Survival + Install the Environment](lessons/a02.md) - Learn to talk to your computer in text, then set up one shared environment (WSL on Windows) and install Node.js. Do the install before class if you can.
- [A03: Install & Authenticate Gemini CLI](lessons/a03.md) - Install Gemini CLI, log in for free with a Google account, and hold your first conversation. No credit card, no API key.
- [A04: How to Ask: Prompting](lessons/a04.md) - The quality of the answer depends on the quality of the question. Be specific, give examples and constraints, and treat the first answer as a draft.
- [A05: Context Engineering](lessons/a05.md) - The AI only knows what you put in front of it. Feed it the actual file with @, keep the context focused, and stop making it guess.
- [A06: Memories with GEMINI.md](lessons/a06.md) - Stop re-explaining yourself every session. A GEMINI.md file gives the assistant standing instructions it reads automatically every time.
- [A07: Custom Commands](lessons/a07.md) - Save your best prompts as reusable shortcuts. Other tools call these skills or agents; in Gemini CLI they are custom commands.
- [A08: Making & Running Scripts with AI](lessons/a08.md) - A script turns a repeated chore into one command. Have the AI write scripts for you, and call the AI from inside a script. Read before you run.
- [A09: Capstone: Your AI Toolkit](lessons/a09.md) - Put it all together. Assemble a personal AI toolkit and prove the one skill that matters most: catching the AI when it is wrong.
