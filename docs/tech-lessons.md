# Technical Lessons

## Phase 1: The Workbench (Environment Setup)

Install the tools before the first cut

> A craftsman prepares the workbench before starting the first piece

- [T01: Environment Setup](lessons/t01.md) - Install VS Code, Node.js, Git, and a browser. Verify with the terminal. Write hello.html and see it run.

## Phase 2: The Command Line (Terminal)

Direct line to the machine - text beats clicks at scale

> Buttons are for tourists. Commands are for residents

- [T02: Terminal Basics](lessons/t02.md) - Navigate, create, delete files. Read the manual. Learn the PATH and speed tricks.
- [T03: Pipes & Power Tools](lessons/t03.md) - stdin/stdout/stderr, pipes, grep/find/sed/awk, chaining, small shell scripts.

## Phase 3: The Book (HTML/CSS)

Structure and decoration, no JavaScript at all

> Like building a book: skeleton (HTML) and design (CSS)

- [T04: Hello World](lessons/t04.md) - Your first webpage. Create a file, open it in a browser.
- [T05: HTML Tags](lessons/t05.md) - The building blocks: headings, paragraphs, lists, links, images.
- [T06: HTML Forms & Blocks](lessons/t06.md) - Interactive HTML: input, select, textarea, validation, details/summary.
- [T07: CSS Basics](lessons/t07.md) - Making things look good: selectors, colors, fonts, box model.
- [T08: CSS Layout](lessons/t08.md) - Organizing the page: flexbox, grid, responsive design, media queries.
- [T09: CSS to the Limit](lessons/t09.md) - Pure CSS interactivity: animations, transitions, :target, accordion/tabs.

## Phase 4: The Brain (JavaScript)

Adding thinking and interaction to the page

> Like adding muscles and a brain to your book

- [T10: JavaScript Intro](lessons/t10.md) - First steps with JS: console.log, variables, types, functions.
- [T11: DOM Manipulation](lessons/t11.md) - Controlling the page: querySelector, createElement, events.
- [T12: Forms & Dialog](lessons/t12.md) - Handling user input: JS form validation, dialog element.
- [T13: Data Structures](lessons/t13.md) - Organizing information: arrays, objects, loops, dynamic lists.
- [T14: Persistence](lessons/t14.md) - Remembering data: localStorage, JSON parse/stringify.
- [T15: Fetch API](lessons/t15.md) - Talking to servers: HTTP requests, async/await, promises.
- [T16: Dynamic Site: Routing](lessons/t16.md) - One page, many views: hash routing, components, state.
- [T17: Dynamic Site: Offline](lessons/t17.md) - Works without internet: service workers, PWA, caching.
- [T18: Dynamic Site: Polish](lessons/t18.md) - Production quality: performance, lazy loading, error handling.

## Phase 5: The Time Machine (Git & GitHub)

Track every change, collaborate without stepping on each other

> Git is your time machine. GitHub is the shared workshop where time travelers meet

- [T19: Git Basics](lessons/t19.md) - The time machine: three areas, commits, branches, undo without fear.
- [T20: GitHub & Collaboration](lessons/t20.md) - The shared workshop: remotes, push/pull, GitHub Flow, pull requests.

## Phase 6: The Kitchen (Backend)

Servers, APIs, and databases

> Building the restaurant kitchen that prepares the food

- [T21: Node.js Server](lessons/t21.md) - Building the kitchen: HTTP server, serving files, request handling.
- [T22: API Endpoints](lessons/t22.md) - The menu: REST design, JSON responses, routing.
- [T23: JSON Database](lessons/t23.md) - The recipe book: read/write files, CRUD with JSON.
- [T24: SQLite](lessons/t24.md) - The filing cabinet: SQL, tables, CRUD, schema, foreign keys.
- [T25: Authentication](lessons/t25.md) - The ID check: sessions, cookies, password hashing, login/logout.

## Phase 7: The Robot Chef (AI)

Integrating artificial intelligence

> Hiring a robot that read millions of books to help in your kitchen

- [T26: Ollama & Chat](lessons/t26.md) - The robot chef: local LLM setup, chat API, chat interface.
- [T27: Full Stack AI App](lessons/t27.md) - The whole restaurant: connect frontend, backend, AI, and database.

## Phase 8: The Franchise Kit (Frontend Frameworks)

Component-based UI and type-safe JavaScript

> Upgrading from a hand-drawn menu to a professional POS system

- [T28: React Foundations](lessons/t28.md) - Component thinking: JSX, props, event handling, rendering lists.

- [T32: Next.js: Data & API](lessons/t32.md) - Data fetching in server components, API routes, building a full page.

## Phase 9: The Headquarters (Backend Frameworks)

## Phase 10: The Lego Kit (Web Components)

Invent your own HTML tags - platform-native components that work in any framework
