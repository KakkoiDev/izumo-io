# T03: パイプ & パワーツール

ターミナルは、パイプを理解した瞬間に不器用なファイルブラウザであることを止めて、超能力になります。各コマンドは小さな専門家。パイプ `|` はダクトテープ: 専門家を組み合わせて組立ラインを作ることができます。パイプで繋いだ5つのコマンドが、スプレッドシートのマクロ、Pythonスクリプト、午後1つ分の作業を置き換えます。
{: .lesson-intro }

## 標準ストリーム

すべてのプログラムは3つのチャネルを持つ。**stdin** は入力ストリーム。**stdout** は通常の出力先。**stderr** はエラーの出力先。シェルは各々を独立にリダイレクトできる。

```
echo "hello" > greet.txt        # stdoutをファイルへ(上書き)
echo "world" >> greet.txt       # stdoutをファイルへ追記
sort < names.txt                 # stdinをファイルから
command 2> errors.log            # stderrをファイルへ
command > out.log 2>&1          # stdoutとstderrをまとめて
command > /dev/null 2>&1         # 全出力を捨てる
```

## パイプ

パイプは1つのコマンドのstdoutを次のコマンドのstdinに繋ぐ。文章のように左から右に読む。

```
# このフォルダで一番大きいファイル5つ
ls -lS | head -n 5

# プロジェクトの.tsファイルを数える
find src -name "*.ts" | wc -l

# コードの中の全てのTODO
grep -rn "TODO" src/ | less

# よく使うコマンドのトップ10
history | awk '{print $2}' | sort | uniq -c | sort -rn | head
```

```mermaid
flowchart LR
    H[history] -->|stdout| A[awk print command]
    A -->|stdout| S1[sort]
    S1 -->|stdout| U[uniq -c]
    U -->|stdout| S2[sort -rn]
    S2 -->|stdout| Hd[head]
```

## 必携の専門家たち

プロが鋭く保つ標準キット:

- `grep pattern file` - パターンに一致する行を探す。`-r` で再帰、`-i` で大文字小文字を無視、`-n` で行番号
- `find path -name "*.ext"` - ファイル名、種別、サイズ、経過時間で探す
- `sort` / `uniq` - 行をソート、重複を畳む(`uniq -c` で数える)
- `wc -l` - 行数を数える
- `head -n 20` / `tail -n 20` - 先頭/末尾の行。`tail -f` は成長するログを追跡

上級: `cut`、`sed`、`awk` でカラム抽出、ストリーム編集、テキスト処理。必要になった時に拾う。

## コマンドのチェイン

3つの演算子でパイプなしにコマンドを順序付ける。

```
cmd1 ; cmd2          # cmd1、続いてcmd2を結果に関わらず実行
cmd1 && cmd2         # cmd1が成功した場合のみcmd2
cmd1 || cmd2         # cmd1が失敗した場合のみcmd2

# 実用例
npm test && git push                     # テストが通ったらpush
mkdir build || echo "already exists"      # フォールバックメッセージ
```

## 小さなシェルスクリプト

同じパイプを毎日実行するなら保存する。シェルスクリプトは `#!/usr/bin/env bash` ヘッダーを持つテキストファイルで、`chmod +x` で実行可能にする。

```
#!/usr/bin/env bash
# backup.sh - プロジェクトをタイムスタンプ付きのtarballにコピー

set -euo pipefail                       # エラーで即失敗

name="$(basename "$PWD")"
stamp="$(date +%Y%m%d-%H%M%S)"
tar -czf "../${name}-${stamp}.tar.gz" .
echo "Saved ../${name}-${stamp}.tar.gz"
```

```
chmod +x backup.sh
./backup.sh                             # 実行
```

## 長時間ジョブの実行

数分かかるジョブもある。監視したくない。

```
long-command &                  # バックグラウンドで実行
jobs                             # このシェルのバックグラウンドジョブを一覧
long-command > out.log 2>&1 &   # バックグラウンド + 出力キャプチャ
ps aux | grep node               # 実行中プロセスを探す
kill 12345                       # プロセスに停止を依頼
kill -9 12345                    # 強制停止
```
