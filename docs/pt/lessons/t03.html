T03: Pipes e Ferramentas de Poder

O terminal deixa de ser um navegador de arquivos desajeitado e vira superpoder no momento em que você entende o pipe. Cada comando é um pequeno especialista. O pipe | é fita adesiva: ele permite parafusar especialistas juntos em linhas de montagem. Cinco comandos unidos por pipes podem substituir uma macro de planilha, um script Python ou uma tarde inteira.

Streams Padrão

Todo programa tem três canais. stdin é o stream de entrada. stdout é para onde vai a saída normal. stderr é para onde vão os erros. O shell permite redirecionar cada um de forma independente.

echo "hello" > greet.txt        # stdout TO a file (overwrite)
echo "world" >> greet.txt       # stdout APPEND to a file
sort < names.txt                 # stdin FROM a file
command 2> errors.log            # stderr TO a file
command > out.log 2>&1          # stdout and stderr together
command > /dev/null 2>&1         # discard all output

O Pipe

Um pipe conecta o stdout de um comando ao stdin do próximo. Você lê da esquerda para a direita, como uma frase.

# Show the 5 largest files in this folder
ls -lS | head -n 5

# Count how many .ts files the project has
find src -name "*.ts" | wc -l

# Find every TODO in your code
grep -rn "TODO" src/ | less

# Top 10 commands you use most
history | awk '{print $2}' | sort | uniq -c | sort -rn | head

Os Especialistas Essenciais

O kit padrão que todo profissional mantém afiado:

  • grep pattern file - encontra linhas que batem com um padrão. Adicione -r para recursão, -i para ignorar maiúsculas, -n para números de linha
  • find path -name "*.ext" - localiza arquivos por nome, tipo, tamanho, idade
  • sort e uniq - ordena linhas, colapsa duplicatas (uniq -c conta)
  • wc -l - conta linhas. Resposta rápida para "quantos?"
  • head -n 20 / tail -n 20 - primeiras / últimas linhas. tail -f segue um log crescendo
  • cut -d, -f1 - extrai uma coluna de dados estilo CSV
  • sed 's/old/new/g' - busca e substituição. Editor de streams para substituições rápidas
  • awk '{print $2}' - processamento de texto ciente de colunas

Encadeando Comandos

Três operadores sequenciam comandos sem pipes.

cmd1 ; cmd2          # run cmd1, then cmd2 regardless of outcome
cmd1 && cmd2         # run cmd2 ONLY if cmd1 succeeded
cmd1 || cmd2         # run cmd2 ONLY if cmd1 failed

# Practical example
npm test && git push                     # push only if tests pass
mkdir build || echo "already exists"      # fallback message

Um Scriptzinho de Shell

Quando você roda o mesmo pipe todo dia, salve. Um shell script é um arquivo de texto com o cabeçalho #!/usr/bin/env bash, feito executável com chmod +x.

#!/usr/bin/env bash
# backup.sh - copy the project into a timestamped tarball

set -euo pipefail                       # fail fast on errors

name="$(basename "$PWD")"
stamp="$(date +%Y%m%d-%H%M%S)"
tar -czf "../${name}-${stamp}.tar.gz" .
echo "Saved ../${name}-${stamp}.tar.gz"

chmod +x backup.sh
./backup.sh                             # run it

Rodando Jobs Demorados

Alguns jobs levam minutos. Você não quer ficar de babá.

long-command &                  # run in background
jobs                             # list background jobs in this shell
long-command > out.log 2>&1 &   # background + capture output
ps aux | grep node               # find running processes
kill 12345                       # ask a process to stop
kill -9 12345                    # force it to stop

Key Takeaways

  • Três streams: stdin, stdout, stderr. Redirecione com <, >, >>, 2>
  • O pipe | conecta o stdout de um comando ao stdin do próximo - leia da esquerda para a direita
  • Kit de poder: grep, find, sort, uniq, wc, head, tail, cut, sed, awk. Aprenda um por dia
  • Encadeie com ; (sempre), && (em sucesso), || (em falha) para comandos multi-passo seguros
  • Salve como shell script qualquer pipe que você rode duas vezes. Torne executável com chmod +x
