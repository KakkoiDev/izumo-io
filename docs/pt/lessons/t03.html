T03: Pipes & Ferramentas Poderosas

O terminal deixa de ser um navegador de arquivos desajeitado e vira um superpoder no momento em que você entende o pipe. Cada comando é um pequeno especialista. O pipe | é fita adesiva: deixa você parafusar especialistas em linhas de montagem. Cinco comandos em pipe podem substituir uma macro de planilha, um script Python, ou uma tarde inteira.

Streams Padrão

Todo programa tem três canais. stdin é o stream de entrada. stdout é onde a saída normal vai. stderr é onde vão os erros. O shell permite redirecionar cada um independentemente.

echo "hello" > greet.txt        # stdout PARA arquivo (sobrescreve)
echo "world" >> greet.txt       # stdout APPEND no arquivo
sort < names.txt                 # stdin DE arquivo
command 2> errors.log            # stderr PARA arquivo
command > out.log 2>&1          # stdout e stderr juntos
command > /dev/null 2>&1         # descarta toda saída

O Pipe

Um pipe conecta a stdout de um comando à stdin do próximo. Leia da esquerda para a direita, como uma frase.

# Os 5 maiores arquivos desta pasta
ls -lS | head -n 5

# Quantos arquivos .ts o projeto tem
find src -name "*.ts" | wc -l

# Todo TODO no código
grep -rn "TODO" src/ | less

# Top 10 comandos que você mais usa
history | awk '{print $2}' | sort | uniq -c | sort -rn | head

Especialistas Essenciais

O kit padrão que todo pro mantém afiado:

  • grep pattern file - acha linhas que batem com um padrão. -r recursivo, -i case-insensitive, -n número da linha
  • find path -name "*.ext" - localiza arquivos por nome, tipo, tamanho, idade
  • sort / uniq - ordena linhas, colapsa duplicatas (uniq -c conta)
  • wc -l - conta linhas
  • head -n 20 / tail -n 20 - primeiras / últimas linhas. tail -f segue um log crescendo

Avançado: cut, sed, awk para extrair coluna, edição em stream, processamento de texto. Pegue sob demanda.

Encadeando Comandos

Três operadores sequenciam comandos sem pipe.

cmd1 ; cmd2          # roda cmd1, depois cmd2 não importa o resultado
cmd1 && cmd2         # roda cmd2 SÓ se cmd1 teve sucesso
cmd1 || cmd2         # roda cmd2 SÓ se cmd1 falhou

# Exemplo prático
npm test && git push                     # push só se testes passarem
mkdir build || echo "already exists"      # mensagem de fallback

Um Script Shell Pequeno

Quando você roda o mesmo pipe todo dia, salve. Um script shell é um arquivo texto com cabeçalho #!/usr/bin/env bash, tornado executável com chmod +x.

#!/usr/bin/env bash
# backup.sh - copia o projeto num tarball com timestamp

set -euo pipefail                       # falha rápido em erros

name="$(basename "$PWD")"
stamp="$(date +%Y%m%d-%H%M%S)"
tar -czf "../${name}-${stamp}.tar.gz" .
echo "Saved ../${name}-${stamp}.tar.gz"

chmod +x backup.sh
./backup.sh                             # rodar

Jobs Longos

Alguns jobs levam minutos. Você não quer babá.

long-command &                  # roda em background
jobs                             # lista jobs de background neste shell
long-command > out.log 2>&1 &   # background + captura saída
ps aux | grep node               # acha processos rodando
kill 12345                       # pede ao processo para parar
kill -9 12345                    # força parar

Pontos-chave

  • Três streams: stdin, stdout, stderr. Redirecione com <, >, >>, 2>
  • O pipe | conecta a stdout de um comando à stdin do próximo - leia da esquerda para a direita
  • Kit de poder: grep, find, sort, uniq, wc, head, tail, cut, sed, awk. Aprenda um por dia
  • Encadeie com ; (sempre), && (no sucesso), || (na falha) para comandos seguros em vários passos
  • Todo pipe que você roda duas vezes, salve como script. Torne executável com chmod +x
