T02: Terminal do Zero

Botões são para turistas. O terminal é a linha direta com a máquina - uma prompt de texto onde você diz ao computador exatamente o que fazer. Por uma semana parece intimidador, depois vira superpoder pelo resto da carreira. Você para de clicar em diálogos e começa a compor comandos.

Onde Estou? Se Movendo

Seu shell está sempre dentro de alguma pasta, chamada diretório de trabalho. Três comandos formam o básico de navegação: onde estou, o que tem aqui, vá para algum lugar.

pwd                    # print working directory
ls                     # list what is in this folder
ls -la                 # long format, include hidden files
cd Documents           # move into a subfolder
cd ..                  # move up one level
cd                     # go home (shorthand for ~)
cd /                   # jump to the filesystem root

Caminhos vêm em dois sabores. Um caminho absoluto começa na raiz: /home/alice/projects. Um caminho relativo começa de onde você está agora: ../notes. Os atalhos especiais . (aqui), .. (pai) e ~ (sua home) aparecem em todo lugar.

Criar, Copiar, Remover

Arquivos e pastas são texto entrando, bytes saindo. Mantenha os verbos curtos.

mkdir my-project              # make a folder
mkdir -p a/b/c                # make nested folders at once
touch notes.md                # create an empty file
cp notes.md backup.md         # copy file
cp -r my-project archive      # copy folder recursively
mv old.md new.md              # rename (or move)
rm notes.md                   # delete file (no recycle bin!)
rm -rf build/                 # delete folder and everything inside
cat notes.md                  # print file to screen
less log.txt                  # scroll through a large file (q to quit)

Aviso: não existe lixeira. rm é final. Antes de usar rm -rf, sempre rode ls primeiro para confirmar o alvo.

Ler o Manual

Todo comando vem com o próprio manual. Quando esquecer uma flag, pergunte.

man ls                 # full manual page (q to quit)
ls --help              # quick summary (most modern commands)
which python           # show where a command lives on disk

O PATH: Por Que os Comandos Simplesmente Funcionam

Quando você digita git, o shell não sabe onde o programa git fica. Ele percorre uma lista de pastas guardada na variável de ambiente PATH e roda o primeiro git que encontra. É por isso que instalar ferramentas às vezes exige atualização do PATH.

echo $PATH                              # see the search list
# /usr/local/bin:/usr/bin:/bin:...
export PATH="$HOME/.local/bin:$PATH"    # add your own folder first

Truques de Velocidade

O shell recompensa a memória muscular. Quatro hábitos que se pagam todo dia:

  • Tab completa nomes de arquivo e comandos. Aperte duas vezes para ver as opções.
  • Seta para cima volta no histórico de comandos.
  • Ctrl+R busca histórico por fragmento - digite "git com" para ressuscitar "git commit -m ...".
  • Ctrl+C cancela um comando em execução. Ctrl+D encerra a entrada / sai do shell.

Pontos-chave

  • pwd, ls, cd são o básico de navegação. Caminhos são absolutos (começam com /) ou relativos (começam daqui)
  • mkdir, touch, cp, mv, rm gerenciam arquivos e pastas. rm é permanente - não existe lixeira
  • man e --help estão sempre aí quando você esquece uma flag. Use antes do Google
  • PATH é a lista de busca que o shell usa para achar comandos. Saiba como inspecionar e estender
  • Autocompletar com Tab, busca de histórico com Ctrl+R e cancelar com Ctrl+C são a memória muscular que separa iniciantes de profissionais
