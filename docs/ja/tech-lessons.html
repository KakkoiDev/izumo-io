技術レッスン

HTMLからフルスタックAIアプリまでの実践レッスン。ステップバイステップで実際のプロジェクトを構築。

フェーズ1: 作業台 (環境セットアップ)

最初の切削の前に道具を揃える

職人は最初の作品を始める前に作業台を整える

T01

環境セットアップ

VS Code、Node.js、ブラウザをインストール。ターミナルで確認。hello.htmlを書いて動かす。

フェーズ2: コマンドライン (ターミナル)

マシンへの直通回線 - 大量作業ではテキストがクリックに勝る

ボタンは観光客向け。コマンドは住人向け

T02

ターミナルの基礎

ナビゲート、作成、削除。マニュアルを読む。PATHとスピード技。

 T03

パイプとパワーツール

stdin/stdout/stderr、パイプ、grep/find/sed/awk、連結、小さなシェルスクリプト。

フェーズ3: 本 (HTML/CSS)

構造と装飾、JavaScriptなし

本を作るように: 骨格 (HTML) とデザイン (CSS)

T04

Hello World

初めてのウェブページ。ファイルを作成し、ブラウザで開く。

 T05

HTMLタグ

基本要素: 見出し、段落、リスト、リンク、画像。

 T06

HTMLフォームとブロック

インタラクティブなHTML: input, select, textarea, バリデーション, details/summary。

 T07

CSS基礎

見た目を整える: セレクタ、色、フォント、ボックスモデル。

 T08

CSSレイアウト

ページの構成: flexbox, grid, レスポンシブデザイン, メディアクエリ。

 T09

CSSの限界に挑む

CSSだけのインタラクティブ: アニメーション, トランジション, :target, アコーディオン/タブ。

フェーズ4: 脳 (JavaScript)

ページに思考とインタラクションを加える

本に筋肉と脳を加える

T10

JavaScript入門

JSの第一歩: console.log, 変数, 型, 関数。

 T11

DOM操作

ページの制御: querySelector, createElement, イベント。

 T12

フォームとダイアログ

ユーザー入力の処理: JSフォームバリデーション, ダイアログ要素。

 T13

データ構造

情報の整理: 配列, オブジェクト, ループ, 動的リスト。

 T14

データの永続化

データの記憶: localStorage, JSON parse/stringify。

 T15

Fetch API

サーバーとの通信: HTTPリクエスト, async/await, Promise。

 T16

動的サイト: ルーティング

1ページで複数のビュー: ハッシュルーティング, コンポーネント, 状態管理。

 T17

動的サイト: オフライン

インターネットなしで動作: サービスワーカー, PWA, キャッシュ。

 T18

動的サイト: 仕上げ

プロダクション品質: パフォーマンス, 遅延読み込み, エラー処理。

フェーズ5: タイムマシン (GitとGitHub)

全ての変更を記録し、他の開発者と衝突せずに協働する

Gitはあなたのタイムマシン。GitHubはタイムトラベラーが集まる共有工房

T19

Gitの基礎

タイムマシン: 3つの領域、コミット、ブランチ、恐れずに取り消す。

 T20

GitHubとコラボレーション

共有工房: リモート、push/pull、GitHub Flow、プルリクエスト。

フェーズ6: 厨房 (バックエンド)

サーバー、API、データベース

料理を準備するレストランの厨房を作る

T21

Node.jsサーバー

厨房を作る: HTTPサーバー, ファイル配信, リクエスト処理。

 T22

APIエンドポイント

メニュー: REST設計, JSONレスポンス, ルーティング。

 T23

JSONデータベース

レシピブック: ファイルの読み書き, JSONでCRUD。

 T24

SQLite

ファイリングキャビネット: SQL, テーブル, CRUD, スキーマ, 外部キー。

 T25

認証

ID確認: セッション, クッキー, パスワードハッシュ, ログイン/ログアウト。

フェーズ7: ロボットシェフ (AI)

人工知能の統合

百万冊の本を読んだロボットを厨房に雇う

T26

Ollamaとチャット

ロボットシェフ: ローカルLLMのセットアップ, チャットAPI, チャットインターフェース。

 T27

フルスタックAIアプリ

レストラン全体: フロントエンド, バックエンド, AI, データベースをつなぐ。

フェーズ8: フランチャイズキット (フロントエンドフレームワーク)

コンポーネントベースUIと型安全なJavaScript

手書きメニューからプロのPOSシステムにアップグレード

T28

React基礎

コンポーネント思考: JSX、Props、イベントハンドリング、リスト描画。

 T29

Reactの状態とエフェクト

インタラクティブUI: useStateで記憶、useEffectで副作用。

 T30

TypeScript

型安全性: 型注釈、インターフェース、Reactコンポーネントの型付け。

 T31

Next.js: ルーティングとレンダリング

ファイルベースルーティング、サーバーvsクライアントコンポーネント、レイアウト。

 T32

Next.js: データとAPI

サーバーコンポーネントでのデータ取得、APIルート、完全なページの構築。

フェーズ9: 本部 (バックエンドフレームワーク)

Nest.jsによるエンタープライズ級のバックエンドアーキテクチャ

全フランチャイズ店を管理する本部キッチンを構築する

T33

Nest.js: アーキテクチャ

モジュール、コントローラー、サービス、依存性注入。

 T34

Nest.js: データと認証

DTO、バリデーション、データベース連携、認証ガード。

フェーズ10: レゴキット (Web Components)

自分のHTMLタグを発明する - どのフレームワークでも動くプラットフォームネイティブなコンポーネント

ブラウザに組み込まれたレゴブロック。フレームワーク不要で組み合わせる

T35

Web Components I

自分のタグを発明: カスタム要素、Shadow DOM、:hostと::part。

 T36

Web Components II

テンプレート、スロット、ライフサイクル、シュガーレイヤーとしてのLit。

フェーズ11: 設計図 (システム設計)

大規模システムの設計: スケール、ストレージ、キャッシング、トレードオフ

建築家はコンクリートを流す前に設計図を描く

T37

システム設計: フレームワーク

要件、エンティティ、API、高レベル設計、ディープダイブ、トレードオフ。

 T38

システム設計: スケールとストレージ

垂直 vs 水平、SQL vs NoSQL、レプリケーション、シャーディング、CAP定理。

 T39

システム設計: キャッシング、キュー、パターン

cache-aside、キュー、ロードバランサ、CDN、モノリス vs マイクロサービス、暗記すべき数字。
