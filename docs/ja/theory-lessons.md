# 理論とライフレッスン

- [R01: ITとは何か？](lessons/r01.md) - すべては情報。ITとは情報の収集、送信、処理、保存。
  - *データの郵便局*
- [R02: ウェブアーキテクチャ](lessons/r02.md) - クライアントがリクエストし、サーバーがレスポンスする。注文を出して厨房が料理を準備するレストランのよう。
  - *レストラン: 客が注文、厨房が準備*
- [R03: 問題解決](lessons/r03.md) - 作る前に理解する。ソフトウェア開発の大半は考えること。
  - *医者: 処方する前に診断する*
- [R04: 20/80の法則](lessons/r04.md) - 20%の知識で実務の80%をカバー。最も重要なことに集中。
  - *5つの調理技法でほとんどの料理をカバー*
- [R05: 継続は情熱に勝る](lessons/r05.md) - 毎日の練習はひらめきに勝る。長い目で見ればマラソンランナーが勝つ。
  - *マラソンランナーはスプリンターに勝つ*
- [R06: 最小抵抗の法則](lessons/r06.md) - 新しいツールは古い方法より簡単な時に採用される。イノベーションは水のように流れる。
  - *水は下り坂を流れる*
- [R07: シンプルに保つ](lessons/r07.md) - シンプルな解決策が複雑なものに勝つ。実務では基本的な工具箱がスイスアーミーナイフに勝る。
  - *工具箱はスイスアーミーナイフに勝る*
- [R08: コード品質](lessons/r08.md) - 良いコードは動き、速く、変更しやすい。レストランのように3つのレベルがある。
  - *レストラン: 食べられる、美味しい、スムーズな厨房*
- [R09: 学び方](lessons/r09.md) - 学ぶとは地図を探索すること。最初は知らないことすら知らない。
  - *未知の地図を探索する*
- [R10: KakkoiSchool ケーススタディ](lessons/r10.md) - このサイトの作り方: Markdownレッスン、HTMLとMarkdownを同時に出力する小さなPythonビルド、翻訳用のGitHub Actions、school.kakkoi.devで配信するGitHub Pages。
  - *毎日使っているものを研究する*
- [R11: 適応力](lessons/r11.md) - IT業界は変化を受け入れる人を評価する。基礎はフレームワークより長持ちする。
  - *基礎はどのツールより長持ちする*
- [R12: ワークライフバランス](lessons/r12.md) - キャリアはマラソン。持続可能なペースが慢性的な過重労働に勝つ。
  - *マラソンを全力疾走はできない*
- [R13: 職場の人間関係](lessons/r13.md) - 職場の政治を乗り越え、仕事を守り、去るべき時を知る。
  - *記録し、味方を作り、集中する*
- [R14: コミュニケーションとチームワーク](lessons/r14.md) - 技術力で採用され、コミュニケーション力で昇進する。
  - *誰にも理解できない優れたコードは無意味*
- [R15: AIとの協働](lessons/r15.md) - AIは力の倍増器。AIに抵抗せず、共に働くことを学ぶ。
  - *百万冊の本を読んだロボットアシスタント*
- [R16: 完成させる力](lessons/r16.md) - コードを書ける人は多いが、完成させられる人は少ない。仕上げと発表はそれ自体がスキル。
  - *完成したプロジェクトは印象的な未完成のものに勝る*
- [R17: 英語の重要性](lessons/r17.md) - 英語は技術世界の共通語。英語を身につけることでリソース、コミュニケーション、キャリアの機会が広がる。
  - *グローバルキッチンの共通言語*
- [R18: ドキュメントは最良の友](lessons/r18.md) - 全てを知る必要はない。仕事は問題を解決すること。ドキュメント、Google、AIを恥ずかしがらず使おう。プライドは邪魔になる。
  - *外科医でさえ手術前に参考書を確認する*
- [R19: ビジネスはお金で動く](lessons/r19.md) - ビジネスはお金で生き延びる。ミッションは顔であり、エンジンではない。チケットの理由を理解し、ビジネスに貢献するコードを書こう。
  - *お金がなければ使命もない*
- [R20: AIを決して信頼するな](lessons/r20.md) - Anthropicの研究で、最先端AIは自己保存のため脅迫し人を死なせる選択をした。大いに使え、決して信じるな。危険操作には人間を置け。
  - *ノコギリは木を切るか手を切るかを気にしない*
- [R21: 技術的エントロピー](lessons/r21.md) - コードは腐る。依存関係は壊れる。DBはデータを失う。自分のスタックも忘れる。唯一の防御はシンプルさ。小さなビルドシステムを伴うMarkdownファイルは10年後も動く。2年前のReactアプリは、しばしば動かない。
  - *庭は草取りをやめた瞬間から自分を食い始める*
